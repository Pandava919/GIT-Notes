1. What is git?

Git is a distributed version control system that tracks versions of files. It is often used to control source code by programmers.

2. What is GitHub?

GitHub is a developer storage platform that allows developers to create, store, manage and share their code and allows developers to work together. It uses Git software, which provides distributed version control of access control, bug tracking, software feature requests, task management, continuous integration for every project.

3. What is git init?

The git init command creates a new Git repository. It can be used to convert an existing, unversioned project to a Git repository or initialize a new, empty repository. Most other Git commands are not available outside of an initialized repository, so this is usually the first command you'll run in a new project.

4. What is git status?

The git status command displays the state of the working directory and the staging area. It lets you see which changes have been staged, which haven't, and which files aren't being tracked by Git.

5. What is git add?

The git add command adds a change in the working directory to the staging area. It tells GIT that you want to include updates to a perticular file in the next commit. git add filename if we want to add all files, git add .

6. What is the use of git commit?

The commit command used to save your changes to the local repository.git commit -m "commit message"

7. What is git restore?

git restore is used to get back the files from staging area. git restore --staged filename also used to restore uncommited changes. git restore filename

8. What is git log?

It is a command to display the committed history.git logto get in one linegit log --oneline

9. What is git remote add?

The git remote add command will create new connection record to a remote repository. git remote add 'name' 'url'

10. What is git branch?

The git branch command is used to create braches, to delete the branches and to list out the available branches. git branch -M master(branch name) git branch 'branch_name' (to create new branch) git branch (to see the branches list) git branch -d 'branch_name'

11. What is git push?

The git push command is used ti upload local repo content to a remote repo. git push -u origin 'branch_name'(setting upstream branch in remote repo). git push Note:
If we are working on different branch we have to change the upstream before pushing code to the remote repo, using command git push -u origin 'branch_name

12. What is git clone?

The git clone is used to copy remote repo to local system. git clone repo_url

13. What is git checkout?

The git checkout is used to change the branch, also we can create and change the branch at a time. git checkout 'branch_name' (switch the branch) git checkout -b 'branch_name' (create and switch the branch)

14. what is git pull?

The git pull is used to take the code from the remote repo to local repo. git pull origin 'branch_name'
